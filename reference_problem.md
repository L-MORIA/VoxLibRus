# Проблема: новый голос Ирины не работает в пайплайне

## 1. Симптом

Пайплайн с `irina_ref.wav` (47 сек, 24 kHz, моно) успешно клонирует голос (Stage 5), но на Stage 6 генерация даёт **1184 секунды (20 минут) аудио на один текстовый чанк** вместо ожидаемых 5–10 секунд.

QA gate бракует: `Too long: 1184.76s > 20.0s`, `Too quiet: RMS=-33.4dB`, `Too much silence: 30.6%`.

## 2. Корень проблемы

### 2.1 F5-TTS использует длительность референса как тайминг-сигнал

F5-TTS_RUSSIAN обучен на парах (референс, текст → аудио). Модель **копирует темп и ритм речи** из референса. Если референс длится 25–47 секунд, модель пытается сгенерировать каждый чанк такой же длительности — даже если в тексте чанка 60 символов.

**Доказательство:**
- `author_ref.wav` (5 сек) → чанки по 6–10 сек ✅
- `irina_ref.wav` (47 сек) → чанки по 1184 сек ❌
- `irina_ref_trimmed.wav` (25 сек) → чанки всё ещё >20 сек ❌

### 2.2 `fix_duration` не переопределяет это поведение

В `cloner.py` добавлен `_calc_fix_duration()`:
```python
dur = len(text) / 12 + 1.0   # ≈ 6 сек для 60 символов
```

Параметр `fix_duration` передаётся в `infer_process()` F5-TTS, где на строке 487:
```python
if fix_duration is not None:
    duration = int(fix_duration * target_sample_rate / hop_length)
```

Но модель, видимо, использует `fix_duration` как **верхнюю границу**, а не как точную длительность. Если reference значительно длиннее `fix_duration`, модель предпочитает длительность референса.

### 2.3 Дополнительный фактор: QA gate перезапускает генерацию

QA gate с порогом `max_duration=20.0s` бракует чанки длиннее 20 секунд и запускает генерацию заново (до 4 попыток). Каждая попытка заново тратит ~24 минуты на 26 batch-ей.

## 3. Что пробовали

| Попытка | Референс | Результат |
|---|---|---|
| `author_ref.wav` | 5 сек | ✅ Работает, чанки 6–10 сек |
| `Ирина 2.wav` (сырой) | 47 сек | ❌ Чанки >1000 сек, QA блокирует |
| `irina_ref_trimmed.wav` | 25 сек | ❌ Чанки всё ещё >20 сек (1184 сек) |
| `irina_ref_short.wav` | 10 сек | ⏳ В процессе |

## 4. Возможные решения

### 4.1 Короткое решение: референс ≤5 секунд
F5-TTS_RUSSIAN ожидает референс 5–15 секунд. Обрезать референс Ирины до **5 секунд** (первая фраза «Здравствуйте! Меня зовут Ира...»).

### 4.2 Среднее решение: починить `fix_duration`
Исследовать, почему `fix_duration` не переопределяет длительность. Возможно, параметр передаётся, но модель игнорирует его, если `cfg.fix_duration` приходит как `None` из pipeline (дефолт `TTSGenerationConfig()`).

### 4.3 Долгосрочное решение: нормализация темпа
Добавить пост-обработку:
- Определить реальный темп речи в референсе (символов/сек)
- Масштабировать длительность каждого чанка под этот темп
- Speed-фактор через F5-TTS (параметр `speed`)

## 5. Рекомендация

Обрезать референс до **5–8 секунд** — первое предложение. Этого достаточно для клонирования тембра, а длительность генерации будет адекватной.
