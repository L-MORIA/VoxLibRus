> **Ревью 1:** 2026-07-18 — Этап 1 (скелет, v0.1.0)
> **Ревью 2:** 2026-07-18 — Этап 2 (`text/` модуль, Pydantic-конфиг, тесты)
> **Метод:** статический анализ кода/конфигов + практическая верификация (запуск кода на подготовленных примерах, сборка пакета, карточки моделей на HuggingFace, официальные обсуждения на GitHub моделей)

## Резюме

Архитектурный документ (`ARCHITECTURE.md`) продуман основательно: обоснованный выбор моделей, чёткое разделение на слои, resume-механизм для многочасовой генерации, внимание к лицензиям.

**Ревью 1** нашло разрыв между документом и кодом (реализовано было ~240 строк из заявленных ~20 модулей), а также одну блокирующую архитектурную ошибку (не тот вариант Qwen3-TTS для клонирования) и один блокирующий баг сборки (`pyproject.toml`). Все P0-пункты из ревью 1 **исправлены** к моменту ревью 2 — подтверждено практически.

**Ревью 2** охватывает реализованный `text/` модуль (`extractor.py`, `cleaner.py`, `chunker.py`, `accents.py`) и переход конфига на Pydantic. Здесь найден новый блокирующий баг — **порядок глав EPUB может не совпадать с порядком чтения** — плюс несколько багов в нормализации текста, подтверждённых прямым запуском кода на сконструированных примерах (не гипотетически).

## 2. Сильные стороны

- Абстракции `ASRInterface` / `TTSInterface` — верное решение, позволяют переключать бэкенды через конфиг без изменения остального кода.
- Resume-механизм заложен в дизайн с самого начала — обязательное требование для генерации на 9+ часов, а не опция "на потом".
- Выбор моделей подкреплён сравнительными таблицами, а не интуицией.
- Осознанное разделение лицензий (Apache-2.0 / CC-BY-NC) на уровне архитектуры.
- `VoiceProfile` как dataclass с `embedding_path`/`meta` — разумно для последующей сериализации голосов.

## 3. Критические проблемы (подтверждены практически)

### 3.1. 🔴 Неверный вариант модели Qwen3-TTS для клонирования голоса

**Текущее состояние (`config.yaml`):**
```yaml
tts:
  qwen3:
    model_id: Qwen/Qwen3-TTS-12Hz-1.7B-CustomVoice
```

**Проблема:** согласно официальной карточке модели на HuggingFace, `CustomVoice`-вариант поддерживает только `generate_custom_voice(text, language, speaker, instruct)`, где `speaker` — один из **9 фиксированных встроенных голосов** (Vivian, Serena, Ryan, Aiden и т.д.). Клонирование из произвольного референсного аудио (`ref_audio` + `ref_text`, от 3 секунд) доступно только в отдельном варианте — **`Qwen3-TTS-12Hz-1.7B-Base`** через метод `generate_voice_clone`.

**Следствие:** основной сценарий продукта ("автор читает 5 минут → клонируем голос") не реализуем с указанной моделью — `TTSInterface.clone_voice()` для Qwen3 нечем будет реализовать.

**Исправление:** заменить `model_id` на `Qwen/Qwen3-TTS-12Hz-1.7B-Base`. Для переиспользования эмбеддинга между чанками без пересчёта — использовать `create_voice_clone_prompt()` из API модели.

### 3.2. 🔴 `pyproject.toml` не собирается

```toml
build-backend = "setuptools.backends._legacy:_Backend"
```

**Проверено:** `pip install -e .` в чистом окружении падает с ошибкой:
```
pip._vendor.pyproject_hooks._impl.BackendUnavailable: Cannot import 'setuptools.backends._legacy'
```
Такого модуля в setuptools нет — это не публичный API. С заменой на `setuptools.build_meta` установка и запуск `voxlib --help` проходят успешно (проверено).

**Исправление:**
```toml
build-backend = "setuptools.build_meta"
```

### 3.3. 🟠 `requirements.txt` неполон для GigaAM-v3

Карточка `ai-sage/GigaAM-v3` требует `pyannote-audio`, `torchcodec`, `hydra-core`, `omegaconf`, `sentencepiece`; модель грузится с `trust_remote_code=True` (исполняется сторонний код — стоит отметить как security-соображение при дистрибуции проекта). В `requirements.txt` этих зависимостей нет — первый же запуск `asr/gigaam.py` упадёт на импортах.

## 4. Специфика русского языка

**Отсутствует расстановка ударений (омографы).** В `cleaner.py` заложены числа→пропись, кавычки, аббревиатуры, "ё" — но не ударения. Это не косметика: *за́мок/замо́к*, *му́ка/мука́* и т.п. модель не разрешит без явной разметки. Даже карточка модели `F5-TTS_RUSSIAN` прямо рекомендует прогонять текст через [RUAccent](https://github.com/Den4ikAI/ruaccent) для приемлемого качества. Стоит добавить этот шаг между `cleaner.py` и `chunker.py`, независимо от TTS-бэкенда.

## 5. Архитектурные риски пайплайна

- **Дрейф голоса на длинных книгах** — риск в таблице помечен, но конкретной митигации (фиксированный seed на профиль, переиспользование `voice_clone_prompt`) в дизайне нет.
- **Нет ревью транскрипции референса перед клонированием.** Сам документ признаёт критичность точности референсного текста, но между `voxlib transcribe` и `voxlib clone` нет шага правки человеком.
- **Whisper-large-v3 как fallback тяжеловесен** для ASR, который используется один раз на 5-10 минут аудио за весь прогон (3B модель, 6-8GB VRAM ради edge-case).
- **Нет аудио-QA перед сборкой** — если TTS сгенерирует "мусорный" сэмпл на каком-то чанке (типичная проблема авторегрессионных TTS, не исключение), пайплайн это не заметит и склеит в финальную книгу.
- **`streaming: true` в конфиге Qwen3 бессмысленно для батч-генерации** книги — стриминг полезен для интерактивного UX, не для оффлайн-обработки 600 чанков подряд.
- **Лицензионное ограничение F5-TTS (CC-BY-NC) не защищено кодом**, только документацией.
- **Нет валидации `config.yaml`** — прямой доступ по ключам словаря без схемы, опечатка уронит выполнение глубоко в рантайме сырым `KeyError`. Плюс класс `Config` не покрывает часть ключей (`chunk_overlap_chars`, `retry_delay`, `mp3_bitrate`, `peak_dbfs` не выведены как properties).
- **Смешение платформенных допущений**: путь `F:/VoxLibRus/` (Windows) в структуре проекта против `source .venv/bin/activate` (Unix) в Quick Start, при том что риск №1 в таблице — "не устанавливается на Windows".

- `streaming: false` для батч-режима генерации.
- Конфиг переведён на Pydantic с типизацией полей.

### 8.2. 🔴 Критично

**(1) EPUB: главы извлекаются в порядке манифеста, а не в порядке чтения.**
`_extract_epub` использует `book.get_items()` — порядок манифеста OPF, а не `<spine>`, который единственный определяет порядок чтения по спецификации EPUB. Подтверждено на сконструированном EPUB (manifest 3-1-2, spine 1-2-3):
```
Порядок глав на выходе extract(): ['Глава 3', 'Глава 1', 'Глава 2']
Ожидаемый порядок (по spine):      Глава 1, Глава 2, Глава 3
```
Риск — перепутанные главы в готовой аудиокниге для EPUB-файлов от конвертеров, где порядки расходятся.
**Фикс:** брать порядок из `book.spine` (список `idref`), резолвить через `get_item_with_id()`.

**(2) `normalize_quotes`: апостроф в слове десинхронизирует состояние кавычек на весь оставшийся текст.**
```
Вход:  ...про Д'Артаньяна. Потом он сказал: "Привет!" А она ответила: "Пока!"
Выход: ...про Д«Артаньяна. Потом он сказал: »Привет!« А она ответила: »Пока!«
```
Апостроф в имени переключает общий флаг open/close, все настоящие кавычки дальше по тексту инвертируются.
**Фикс:** апостроф внутри слова (без пробела вокруг) не считать кавычкой; кавычки нормализовать регуляркой с учётом границ, не единым stateful-проходом по всем символам-кандидатам сразу.

**(3) `expand_abbreviations`: замена без границ слова портит обычные слова.**
```
Вход:  Т.д. и т.п. существует множество примеров. Также см. приложение.
Выход: Т.д. и прочее существует множество примеровек Также смотри приложение.
```
Сокращение `"в."` → `"век"` матчится внутри `"примеров."` (любое слово на "в" перед точкой конца предложения) — `str.replace` без учёта границ слова. Заодно съедает точку конца предложения, что бьёт по нарезке на предложения в `chunker.py`.
**Фикс:** `re.sub` с проверкой границы слова, например `(?<![а-яёА-ЯЁ])в\.(?=\s|$)`.

**(4) `expand_abbreviations`: регистрозависимость.**
Тот же прогон показывает: `"Т.д."` (заглавная, как бывает в начале предложения) не разворачивается — словарь содержит только нижний регистр, `str.replace` регистрозависим.
**Фикс:** `re.IGNORECASE` + сохранение регистра первой буквы в замене.

**(5) `normalize_numbers`: суффиксы, проценты, годы.**
```
Вход:  ...24-летие, а не 5-й раз... Скидка 50% на всё. Родился в 1990 году.
Выход: ...двадцать четыре-летие, а не пять-й раз... Скидка пятьдесят% на всё. ...в одна тысяча девятьсот девяносто году.
```
Дефисные порядковые суффиксы не обрабатываются («пять-й»), `%` не разворачивается в слово, `num2words` не делает падежную морфологию («в одна тысяча девятьсот девяносто году» вместо «в тысяча девятьсот девяностом году»). Последнее — системное ограничение библиотеки, не мгновенно фиксится, но `%`/суффиксы стоит закрыть отдельными правилами.

**(6) Чанкер: overlap может резать слово пополам.**
```
chunk 1: ...'потока слов подряд не'
chunk 2: 'три этого потока слов подряд не...'
```
`remaining[break_pos - overlap:]` — плоский сдвиг на 50 символов без проверки, что позиция не разрезает слово («внутри» → «вну» / «три»).
**Фикс:** искать ближайший пробел на позиции `break_pos - overlap`, не резать по индексу вслепую. Отдельно нужно решить на уровне дизайна: overlap — это дублирующийся в озвучке текст (тогда при склейке в `assemble.py` будет слышно повторение — «заикание» книги) или только текстовый контекст для TTS (тогда `TTSInterface` должен уметь отличать «контекст» от «текста к озвучке», сейчас не умеет). Решить это нужно до реализации `audio/assemble.py`.

### 8.3. 🟠 Средняя серьёзность

**(7)** `chunk_text(max_chars=...)` не передаётся в `_split_chapter_into_segments` — решение «разбивать абзац на предложения или нет» принимается по захардкоженной `DEFAULT_MAX_CHARS=1000`, а не по параметру функции.

**(8)** Pydantic-валидация не ловит опечатки в ключах YAML (`extra` по умолчанию `"ignore"`):
```python
raw['tts']['primry'] = 'qwen3'   # опечатка вместо 'primary'
del raw['tts']['primary']
Config(**raw)   # → загружается без единой ошибки, tts.primary тихо = дефолт 'qwen3'
```
**Фикс:** `model_config = ConfigDict(extra="forbid")` в моделях.

**(9)** `accents.py`: `t.join(timeout=8.0)` не отменяет фоновый поток — если загрузка RUAccent завершится позже таймаута, глобальное состояние `_accentizer_available` изменится асинхронно без синхронизации, часть чанков книги может озвучиться без ударений, часть — с ними, без предупреждения в логах.

**(10)** Не подтверждено, что primary-бэкенд (Qwen3-TTS-Base) понимает `+`-нотацию ударений от RUAccent. Это конвенция, которую явно поддерживает `F5-TTS_RUSSIAN`, но в [официальном обсуждении на GitHub Qwen3-TTS](https://github.com/QwenLM/Qwen3-TTS/discussions/53) поддержка контроля ударений для русского — открытый фиче-реквест, а не подтверждённая возможность. Стоит проверить на реальном инференсе перед тем, как полагаться на модуль в проде.

**(11)** `requirements.txt`: `markitdown` без экстра-зависимости `[docx]` — воспроизведено тестом:
```
FAILED tests/test_extractor.py::TestSupportedFormats::test_extract_docx_returns_dict
DocxConverter threw MissingDependencyException — pip install markitdown[docx]
```

### 8.4. 🟡 Мелочи

- `_SENTENCE_BOUNDARY` в `chunker.py` объявлена и нигде не используется (мёртвый код).
- Смешение `warnings.warn` (config.py) и `logging` (accents.py) для предупреждений — стоит унифицировать.
- `test_nested_quotes_not_supported` в `test_cleaner.py` проверяет только «не падает», не «работает правильно» — баг с апострофами (8.2.2) в принципе не мог быть пойман текущим набором тестов.

from voxlib.text.cleaner import expand_abbreviations
print(expand_abbreviations('Т.д. и т.п. существует множество примеров. Также см. приложение.'))
# → 'Т.д. и прочее существует множество примеровек Также смотри приложение.'

# → 'Ему было двадцать четыре-летие, а не пять-й раз это заметили. '
#   'Скидка пятьдесят% на всё. Родился в одна тысяча девятьсот девяносто году.'

# 5) Overlap режет слово пополам
from voxlib.text.chunker import chunk_text

        'посередине потока символов...')

# второй чанк начнётся с обрывка слова, например 'три этого потока...'
#   (исходное слово 'внутри' разрезано на 'вну' + 'три')
