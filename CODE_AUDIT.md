# Аудит кода VoxLibRus

Дата: 18 июля 2026 года. Область проверки: весь код и конфигурация в ветке
`master`, упаковка Python и автоматические тесты. Проверка не включала запуск
моделей ASR/TTS: в репозитории пока нет их реализаций.

## Исправлено в этом изменении

| Приоритет | Проблема | Решение |
| --- | --- | --- |
| P1 | Установка пакета не содержала библиотек, которые импортирует текстовый слой; спецификация `markitdown` была некорректной. | Дополнены `pyproject.toml` и `requirements.txt`, включён DOCX extra. |
| P1 | DOCX извлечение падало, когда MarkItDown установлен без DOCX-extra, хотя запасной путь был заявлен. | Добавлен рабочий fallback на `python-docx`. |
| P1 | Chunker мог выдавать фрагмент длиннее `max_chars`; некорректные параметры могли привести к бесконечному циклу. | Добавлена валидация и гарантия продвижения при overlap. |
| P1 | EPUB и DOCX теряли главы с совпадающими заголовками из-за перезаписи ключа словаря. | Добавлено безопасное присвоение уникального суффикса. |
| P2 | Пустой YAML приводил к неясной ошибке TypeError. | Пустая конфигурация теперь использует значения по умолчанию, неверный корень отклоняется явно. |
| P2 | Тест перезаписывал отслеживаемый бинарный EPUB fixture. | Fixture создаётся во временной директории. |

## Открытые риски и рекомендуемый порядок работ

| Приоритет | Наблюдение | Рекомендация |
| --- | --- | --- |
| P0 | Команды CLI (`run`, `transcribe`, `clone`, `generate`, `assemble`) — заглушки, конкретных ASR/TTS/audio backend-ов нет. | Сначала реализовать сквозной минимальный pipeline: один ASR backend, один TTS backend, менеджер прогресса и возобновления; затем добавлять альтернативные модели. |
| P1 | Концепция допускает загрузку моделей с удалённым кодом и обработку недоверенных книг. | Явно закреплять revision/commit моделей, требовать явное согласие на `trust_remote_code`, ограничить размер и распакованный объём PDF/EPUB, запускать конвертацию в изолированном процессе. |
| P1 | Голосовое клонирование не сопровождается проверкой согласия, журналом происхождения образца или правилами удаления данных. | Добавить явный флаг согласия, метаданные профиля (автор, дата, источник, срок хранения) и команду удаления профиля. |
| P1 | README заявляет Apache-2.0, но файла `LICENSE` нет; F5-TTS указан как некоммерческий. | Добавить `LICENSE` после подтверждения правообладателя и разделить/маркировать non-commercial backend, чтобы не создавать ложного впечатления о лицензии всего продукта. |
| P2 | Нет CI, форматтера, type-checker, dependency lockfile и сканирования зависимостей. | Добавить GitHub Actions с `pytest`, `ruff`, `mypy` и Dependabot; зафиксировать воспроизводимые зависимости для поддерживаемых Python 3.11–3.12. |
| P2 | Тесты покрывают текстовый слой, но нет тестов CLI, реального аудио-конвейера, восстановления после сбоя и качества синтеза. | Добавить contract-тесты backend-ов, smoke-тест с маленькой моделью и интеграционный тест resume. |
