## 2. План устранения

### Задача 1: Склейка чанков (неестественные паузы)

**Причина:** Между каждым chunk-ом вставляется `chapter_pause_sec: 2.5`. Текст разбивается на 14 chunks — между ними 13 пауз по 2.5 сек = 32.5 сек тишины. В живой речи таких пауз нет.

**Решение:**

```python
# assemble.py — вставлять паузу ТОЛЬКО между chapter-файлами, не между chunk-ами
# Сейчас: chunk → пауза → chunk → пауза → chunk (все уровни)
# Нужно:  chapter1 (chunks) → пауза → chapter2 (chunks)
```

**План:**
- [ ] Разделить chunk-файлы по chapter-ам (уже есть в структуре temp/*/chapters/)
- [ ] Склеивать chunks внутри chapter без пауз (concat без silence)
- [ ] Вставлять паузу 2.5 сек только между chapter-ами
- [ ] Тест: звуковой файл без лишних пауз

**Сложность:** 🟢 Лёгкая (1-2 часа)

---

### Задача 2: Звуковой мусор (normalize + компрессия)

**Причина:** 
- `target_lufs: -16` — агрессивная нормализация тянет шум
- F5-TTS генерирует чистый аудиосигнал, но normalize повышает усиление и шум тоже

**Решение:**

```yaml
# config.yaml — мягкая нормализация
audio:
  target_lufs: -20      # Было -16 — мягче, меньше артефактов
  peak_dbfs: -3          # Было -1 — больше headroom
```

```python
# normalize.py — добавить noise gate перед нормализацией
# Отсекать тишину ниже порога (например, -50 dB) 
# чтобы шум не усиливался
```

**План:**
- [ ] Снизить target_lufs до -20
- [ ] Добавить noise gate в normalize (отсечка ниже -50 dB)
- [ ] Тест: сравнить «до/после» на спектрограмме

**Сложность:** 🟢 Лёгкая (1 час)

---

### Задача 3: Нестабильный темп

**Причина:** F5-TTS `infer_process()` использует автоопределение длительности на основе текста. Для коротких фраз (<10 символов) использует `speed=0.3`, для длинных — `speed=1.0`. Резкий перепад.

**Решение:**

```python
# f5tts.py — передавать fix_duration в infer_process
# Рассчитывать длительность по количеству символов:
# средняя скорость речи ~10 символов/сек
# fix_duration = len(gen_text) / 10 + 1.0  # +1 сек запас
```

**План:**
- [ ] Отключить автоопределение длительности
- [ ] Рассчитывать `fix_duration` по формуле
- [ ] Тест: проверить что темп равномерный

**Сложность:** 🟡 Средняя (2-3 часа)

---

### Задача 4: Нечёткость звука (Vocos качество)

**Причина:** Vocos (vocoder) — лёгкая модель для mel→audio. Качество ограничено размером модели.

**Решение (варианты):**

| Вариант | Сложность | Эффект |
|---|---|---|
| A. BigVGAN v2 вместо Vocos | 🟡 Средняя | Выше качество, больше памяти |
| B. Увеличить target_sample_rate до 44100 | 🟢 Лёгкая | Выше детализация |
| C. Отключить normalize для чистоты | 🟢 Лёгкая | Меньше артефактов |

**План:**
- [ ] Попробовать BigVGAN v2 (уже есть поддержка в f5_tts)
- [ ] Сравнить спектрограммы Vocos vs BigVGAN
- [ ] Опционально: 44.1 kHz выход

**Сложность:** 🟡 Средняя (3-4 часа)

---

### Задача 5: Ошибки ударений

**Причина:** RUAccent `tiny` модель (самая лёгкая) — меньше параметров, выше вероятность ошибки.

**Решение:**

```python
from ruaccent import RUAccent
accentizer = RUAccent()
# Вместо tiny — использовать turbo или medium_poetry
accentizer.load(omograph_model_size='turbo3.1')
```

| Модель | Размер | Качество |
|---|---|---|
| tiny | ~30 MB | 🟢 База |
| turbo | ~100 MB | 🟡 Лучше |
| turbo3.1 | ~150 MB | 🟡 Ещё лучше |
| medium_poetry | ~300 MB | 🔴 Тяжёлая |
| big_poetry | ~500 MB | 🔴 Очень тяжёлая |

**План:**
- [ ] Переключиться на `turbo3.1` (best balance)
- [ ] Сделать fallback: turbo3.1 → tiny если OOM
- [ ] Тест: сравнить ударения на сложных словах

**Сложность:** 🟢 Лёгкая (30 мин)

## 3. Требования к эталонному образцу голоса (reference audio)

### 3.1 Зачем нужен качественный образец

Качество клонирования голоса напрямую зависит от reference audio. F5-TTS учится по вашему образцу:
- Тембр, интонации, ритм — копируются из референса
- Чистота записи → чистота генерации
- Разнообразие фонем →自然ность речи

### 3.2 Технические параметры

| Параметр | Минимум | Рекомендуется | Максимум |
|---|---|---|---|
| **Формат** | WAV / FLAC | **WAV** | FLAC 24-bit |
| **Разрядность** | 16 bit | **16 bit** | 24 bit |
| **Частота дискретизации** | 16000 Hz | **24000 Hz** | 48000 Hz |
| **Каналы** | 1 (моно) | **1 (моно)** | 1 (моно) |
| **Длительность** | 10 сек | **20-30 сек** | 60 сек |
| **Уровень сигнала** | -6 dB peak | **-3 dB peak** | -1 dB peak |
| **Шум** | SNR > 30 dB | **SNR > 40 dB** | — |
| **Отсечения (clipping)** | 0% | **0%** | 0% |

### 3.3 Рекомендации по записи

- **Микрофон:** любой качественный (Shure, Rode, Blue Yeti, или встроенный микрофон ноутбука в тихой комнате)
- **Помещение:** без эха, без фонового шума (закрытая комната с ковром/мягкой мебелью)
- **Расстояние до микрофона:** 15-30 см
- **Громкость:** говорить в комфортном темпе, без напряжения
- **Тишина:** паузы между предложениями оставлять естественные

### 3.4 Текст образца

Текст должен покрывать **все основные фонемы русского языка** и различные интонационные конструкции.

**Требования к тексту:**
- Длина: 60-80 слов (~20-30 секунд речи)
- Все гласные: а, о, у, э, ы, и, е, ё, ю, я
- Твёрдые и мягкие согласные
- Шипящие: ж, ш, ч, щ
- Сонорные: р, л, м, н
- Вопросительные и восклицательные интонации
- Паузы между предложениями

### 3.5 Пример текста (рекомендуемый)

```
— Здравствуйте! Меня зовут [Ваше Имя].
Я прочитаю несколько предложений для настройки голосового синтезатора.

Вот уже полгода я живу в этом маленьком городке у моря.
Каждое утро я выхожу на берег и смотрю на волны.
Чайки кружатся над головой и кричат что-то своё.

Вы знаете, что корабли уходят в плавание даже в шторм?
Их ведут отважные капитаны. А туман рассеивается только к обеду.

Железная дорога проходит через весь полуостров.
Путь от станции до дома занимает примерно двадцать минут.
Я часто вспоминаю нашу первую встречу — это было чудесно!

Ещё раз: здравствуйте. Меня зовут [Ваше Имя].
```

**Почему этот текст:**
| Элемент | Зачем |
|---|---|
| «Здравствуйте» | Длинное слово, все согласные |
| «полгода», «маленьком» | Мягкие/твёрдые Л, Г |
| «море», «утро», «волны» | Сонорные Р, Л, М, Н |
| «чайки», «кричат» | Шипящие Ч, Щ |
| «знаете», «уходят» | Разные гласные |
| «шторм», «отважные» | Стечение согласных |
| «капитаны», «туман» | Чередование ударных/безударных |
| «железная», «полуостров» | Сложные слова |
| «двадцать минут» | Числительные |
| ? ! — «тире» | Разная интонация |

### 3.6 Альтернатива: тестовый набор пословиц и скороговорок

Для максимального охвата фонем можно использовать:

```
— Карл у Клары украл кораллы, а Клара у Карла украла кларнет.
— На дворе трава, на траве дрова.
— Шла Саша по шоссе и сосала сушку.
— Ехал Грека через реку, видит Грека — в реке рак.
— Съел молодец тридцать три пирога с пирогом, да всё с творогом.
— Везёт Сенька Саньку с Сонькой на санках.
— Цапля чахла, цапля сохла, цапля сдохла.
— Разнервничавшегося конституционалиста нашли акклиматизировавшимся.
```

**Этот вариант сложнее для чтеца**, но даёт наилучший охват фонем.

### 3.7 Проверка качества образца

После записи проверьте:
1. **Нет clipping** —波形 не упирается в 0 dB
2. **Нет фонового шума** — тишина между фразами должна быть чистой (> -50 dB)
3. **Нет эха/реверберации**
4. **Ровный уровень громкости** — без резких перепадов
5. **Естественный темп** — не слишком быстро, не слишком медленно

**Программа для проверки:** Audacity (бесплатно) — открыть файл, визуально проверить波形.
