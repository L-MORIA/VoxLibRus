# RUAccent: ONNX token_type_ids баг — диагностика и план исправления

## 1. Симптом

В логе пайплайна (Stage 3):
```
Accent placement failed: Required inputs (['token_type_ids']) are missing
from input feed (['input_ids', 'attention_mask']).
```

После этого `fix_accents()` возвращает оригинальный текст **без ударений**. Пайплайн пишет «Stress marks added», но на самом деле текст проходит без изменений.

Пользователь слышит ошибки в ударениях (вернулись, хотя раньше были лучше).

## 2. Диагностика

### 2.1 Версии

| Пакет | Версия |
|---|---|
| `ruaccent` | 1.5.8.3 |
| `onnxruntime` | 1.27.0 |

### 2.2 Что работает, что нет

| Вход | Длина | Результат |
|---|---|---|
| `замок` | 5 символов | ✅ `з+амок` |
| `замок стоит на холме` | 20 символов | ✅ `з+амок ст+оит на холм+е` |
| `привет` | 6 символов | ❌ `token_type_ids` missing |
| `Шерлок Холмс...` | 40+ символов | ❌ `token_type_ids` missing |
| Любая фраза с пробелами > 20 символов | >20 | ❌ `token_type_ids` missing |

### 2.3 Корень

ONNX модели `turbo3.1` экспортированы со старой версией ONNX opset, где `token_type_ids` был частью графа. Современный `onnxruntime 1.27.0` обрабатывает `token_type_ids` иначе — для некоторых BPE-токенов runtime требует этот вход, хотя модель его не использует.

Проблема проявляется не на длине текста, а на **конкретных токенах** (BPE-токенизация). «привет» ломается, «замок» работает — хотя оба короткие.

### 2.4 Почему sentence-by-sentence не помогает

Даже если бить текст на отдельные предложения, некоторые предложения содержат «проблемный» токен и падают целиком. Для русского текста со средним разнообразием лексики падает ~60-80% предложений.

## 3. Варианты исправления

### Вариант A: Понизить onnxruntime ⭐ (рекомендуется)

**Суть:** установить onnxruntime версии 1.15-1.17 (до breaking change)

```bash
pip install onnxruntime==1.17.1
```

**Плюсы:** чинит проблему на корню, без изменения кода
**Минусы:** 
- Более старая версия может иметь неисправленные баги
- Другие зависимости могут требовать 1.27+

**Вердикт:** самый простой вариант, вероятность успеха ~90%

### Вариант B: Обновить ruaccent

**Суть:** установить последнюю версию ruaccent

```bash
pip install ruaccent --upgrade
```

**Плюсы:** автор мог починить совместимость с новым onnxruntime
**Минусы:**
- может не быть обновления (версия 1.5.8.3 уже актуальная)
- новое onnxruntime могло изменить API несовместимо

### Вариант C: Изолировать ruaccent в отдельный процесс

**Суть:** вызывать RUAccent через subprocess с Python 3.11 (где установлена старая onnxruntime)

**Плюсы:** не трогает существующее окружение
**Минусы:** сложно, медленно (каждый вызов — новый процесс)

### Вариант D: Заменить RUAccent на другую библиотеку ударений

**Суть:** альтернативные решения
- `pymorphy3` — только морфология, ударений нет
- `syllabus` — слоговая сегментация, не ударения
- `russian_stress` — другая библиотека

**Плюсы:** независимость от ONNX
**Минусы:** требуется интеграция, качество неизвестно

### Вариант E: Свой sentence-by-sentence с пост-валидацией

**Суть:** делать fix_accents по предложениям, но проверять,
что в результате есть `+`, и если нет — ретраить с `tiny` моделью

**Минусы:** всё равно часть предложений будет без ударений

## 4. Рекомендуемый план

1. **Попробовать onnxruntime 1.17.1** (Вариант A) — 5 минут, если работает — чиним
2. Если не помогло — **обновить ruaccent** (Вариант B)
3. Если всё ещё не работает — **заменить библиотеку** (Вариант D)
